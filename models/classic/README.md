# Классические методы машинного обучения

Описание трёх классических методов, используемых для прогнозирования прочности бетона на сжатие.

---

## 1. Линейная регрессия (`model_linear.py`)

### Что это
Линейная регрессия — это самый простой метод машинного обучения для задач прогнозирования. Модель ищет линейную зависимость между входными признаками (состав бетона) и целевой переменной (прочность).

### Как работает
Модель строит уравнение вида:

```
Прочность = w1*Цемент + w2*Шлак + w3*Зола + w4*Вода + w5*Добавка + w6*Крупный_щебень + w7*Мелкий_щебень + w8*Возраст + b
```

где `w1...w8` — веса (коэффициенты), а `b` — свободный член (смещение).

Во время обучения модель подбирает такие значения весов, чтобы сумма квадратов отклонений предсказаний от реальных значений была минимальной (метод наименьших квадратов).

### Плюсы
- Простота и понятность
- Быстрое обучение
- Легко интерпретировать результаты (видно какой признак сильнее влияет)

### Минусы
- Не улавливает нелинейные зависимости
- Чувствительна к выбросам в данных
- Низкая точность на сложных данных

### Библиотека
`sklearn.linear_model.LinearRegression`

## 2. Лассо и гребневая регрессия (`model_lasso_ridge.py`)

### Что это
Это модификации линейной регрессии с добавлением **регуляризации** — штрафа за слишком большие веса. Регуляризация помогает бороться с переобучением (когда модель запоминает обучающие данные вместо того, чтобы находить общие закономерности).

### Как работает

Обе модели решают ту же задачу, что и линейная регрессия, но с дополнительным условием:

**Гребневая регрессия (Ridge)** — добавляет штраф L2:
```
Потери = Сумма_квадратов_ошибок + alpha * (w1² + w2² + ... + w8²)
```
Штраф уменьшает все веса, но не обнуляет их. Параметр `alpha` контролирует силу штрафа.

**Лассо-регрессия (Lasso)** — добавляет штраф L1:
```
Потери = Сумма_квадратов_ошибок + alpha * (|w1| + |w2| + ... + |w8|)
```
Штраф может обнулять отдельные веса, фактически исключая неважные признаки из модели. Это делает лассо инструментом **отбора признаков**.

### Разница между ними
| Свойство | Гребневая (Ridge) | Лассо (Lasso) |
|---|---|---|
| Тип штрафа | L2 (квадраты весов) | L1 (модули весов) |
| Что делает с весами | Уменьшает все | Может обнулить некоторые |
| Отбор признаков | Нет | Да |
| Когда лучше | Все признаки важны | Есть лишние признаки |

### Плюсы
- Защита от переобучения
- Лассо автоматически отбирает важные признаки
- Стабильнее линейной регрессии на зашумлённых данных

### Минусы
- Всё ещё линейная модель — не ловит сложные зависимости
- Нужно подбирать параметр `alpha`

### Библиотеки
- `sklearn.linear_model.Lasso`
- `sklearn.linear_model.Ridge`

## 3. Random Forest (`model_random_forest.py`)

### Что это
Random Forest (случайный лес) — это ансамблевый метод, который строит множество деревьев решений и усредняет их предсказания. Это один из самых мощных и популярных классических алгоритмов.

### Как работает

**Шаг 1: Создание деревьев.**
Алгоритм строит 100 деревьев решений (в нашем случае). Каждое дерево — это набор правил вида:
```
Если Цемент > 300 и Возраст > 28 и Вода < 180:
    Прочность = 55.3 МПа
```

**Шаг 2: Случайность.**
Каждое дерево обучается на случайной подвыборке данных (бутстрэп) и на случайном подмножестве признаков. Это делает деревья разными и независимыми друг от друга.

**Шаг 3: Усреднение.**
Для предсказания каждое дерево даёт свой ответ, а итоговый результат — это среднее всех 100 ответов:
```
Итоговая_прочность = (Дерево_1 + Дерево_2 + ... + Дерево_100) / 100
```

### Почему это работает хорошо
- Каждое отдельное дерево может ошибаться, но ошибки разных деревьев компенсируют друг друга при усреднении
- Деревья решений по природе хорошо улавливают нелинейные зависимости
- Не требует масштабирования данных

### Плюсы
- Высокая точность на табличных данных
- Устойчивость к выбросам и шуму
- Не требует нормализации данных
- Показывает важность признаков
- Практически не переобучается при достаточном числе деревьев

### Минусы
- Работает медленнее линейных моделей
- Сложно интерпретировать (100 деревьев — это "чёрный ящик")
- Потребляет больше памяти

### Библиотека
`sklearn.ensemble.RandomForestRegressor`

## Сравнение методов

| Критерий | Линейная регрессия | Лассо/гребневая | Random Forest |
|---|---|---|---|
| Сложность модели | Простая | Простая | Сложная |
| Нелинейные зависимости | Нет | Нет | Да |
| Защита от переобучения | Нет | Да (регуляризация) | Да (ансамбль) |
| Скорость обучения | Мгновенно | Мгновенно | Средняя |
| Интерпретируемость | Высокая | Высокая | Низкая |
| Точность на данных бетона | Низкая (~0.63) | Низкая (~0.63) | Высокая (~0.89) |

## Общие метрики оценки

- **R² (коэффициент детерминации)** — доля дисперсии, объяснённая моделью. 1.0 = идеально, 0.0 = модель не лучше среднего значения
- **MSE (среднеквадратичная ошибка)** — среднее квадратов отклонений. Чем меньше, тем лучше
- **RMSE** — корень из MSE, в тех же единицах что и целевая переменная (МПа)
- **MAE (средняя абсолютная ошибка)** — среднее модулей отклонений. Более устойчива к выбросам чем MSE
