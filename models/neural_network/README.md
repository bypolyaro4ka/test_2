# Нейронные сети (Keras/TensorFlow)

Описание трёх нейронных сетей, используемых для прогнозирования прочности бетона на сжатие. Каждая сеть является аналогом соответствующего классического метода.

---

## Что такое нейронная сеть

Нейронная сеть — это математическая модель, вдохновлённая работой биологического мозга. Она состоит из слоёв **нейронов**, соединённых между собой **весами**. Каждый нейрон принимает входные данные, умножает их на веса, суммирует и пропускает результат через **функцию активации**.

### Общая схема работы

```
Входные данные (8 признаков бетона)
        ↓
  [ Скрытые слои ]  ← нейроны находят закономерности
        ↓
  Выходной нейрон   ← выдаёт предсказанную прочность (МПа)
```

### Как обучается нейронная сеть

1. **Прямой проход** — данные проходят через все слои, сеть выдаёт предсказание
2. **Подсчёт ошибки** — предсказание сравнивается с реальным значением через функцию потерь (MSE)
3. **Обратное распространение** — алгоритм вычисляет, как нужно изменить каждый вес, чтобы уменьшить ошибку
4. **Обновление весов** — оптимизатор Adam корректирует веса на маленький шаг в нужном направлении
5. **Повторение** — шаги 1-4 повторяются 200 раз (эпох) на всех обучающих данных

### Ключевые понятия

- **Dense (полносвязный слой)** — каждый нейрон связан со всеми нейронами предыдущего слоя
- **ReLU (функция активации)** — пропускает положительные значения как есть, а отрицательные заменяет на ноль. Позволяет сети находить нелинейные зависимости
- **Adam (оптимизатор)** — алгоритм обновления весов, который адаптирует скорость обучения для каждого веса отдельно
- **MSE (функция потерь)** — среднеквадратичная ошибка, которую сеть стремится минимизировать
- **Эпоха** — один полный проход по всем обучающим данным

## 1. Линейная регрессия NN (`model_nn_simple.py`)

### Замена: Линейная регрессия

### Архитектура
```
Вход (8 признаков) → Dense(64, ReLU) → Dense(1)
```

### Что это
Самая простая нейронная сеть с одним скрытым слоем из 64 нейронов. По аналогии с линейной регрессией — это базовая модель без дополнительных усложнений.

### Как работает
1. Входной слой принимает 8 признаков бетона
2. Единственный скрытый слой из 64 нейронов с активацией ReLU находит базовые зависимости между признаками
3. Выходной нейрон без активации выдаёт числовое значение прочности

### Почему это аналог линейной регрессии
- Минимальная архитектура без дополнительных механизмов
- Один слой — простейшее нелинейное расширение линейной модели
- Если бы не было ReLU, сеть работала бы как обычная линейная регрессия

### Параметры модели
- **Количество параметров:** 641 (576 весов + 65 смещений)
- **Эпох обучения:** 200
- **Размер батча:** 32
- **Оптимизатор:** Adam (lr=0.001)

## 2. Лассо/гребневая регрессия NN (`model_nn_regularized.py`)

### Замена: Лассо и гребневая регрессия

### Архитектура
```
Вход (8 признаков) → Dense(128, ReLU, L2=0.001) → Dense(64, ReLU, L2=0.001) → Dense(1)
```

### Что это
Нейронная сеть с двумя скрытыми слоями и **L2-регуляризацией** на весах. По аналогии с лассо/гребневой регрессией — модель со встроенной защитой от переобучения через штрафы на веса.

### Как работает
1. Входной слой принимает 8 признаков
2. Первый скрытый слой (128 нейронов) находит первичные зависимости
3. Второй скрытый слой (64 нейрона) комбинирует найденные паттерны
4. К весам обоих слоёв добавлен штраф L2 (lambda=0.001)

### Что такое L2-регуляризация в нейронной сети
При обучении к основной ошибке добавляется штраф за большие веса:
```
Итоговые потери = MSE + 0.001 * (сумма квадратов всех весов)
```
Это заставляет сеть использовать маленькие веса, что делает модель более «гладкой» и устойчивой. Принцип идентичен гребневой регрессии (Ridge), где тоже используется L2-штраф.

### Почему это аналог лассо/гребневой регрессии
- L2-регуляризация в нейронной сети — это тот же принцип, что и в гребневой регрессии
- Штраф не даёт весам расти слишком большими
- Модель не запоминает шум в данных, а находит общие закономерности

### Параметры модели
- **Количество параметров:** 9,473
- **Регуляризация:** L2 (lambda=0.001)
- **Эпох обучения:** 200
- **Размер батча:** 32
- **Оптимизатор:** Adam (lr=0.001)

## 3. Random Forest NN (`model_nn_deep.py`)

### Замена: Random Forest Regressor

### Архитектура
```
Вход (8 признаков) → Dense(256, ReLU) → Dropout(0.2) → Dense(128, ReLU) → Dropout(0.2) → Dense(64, ReLU) → Dense(1)
```

### Что это
Глубокая нейронная сеть с тремя скрытыми слоями и слоями **Dropout**. По аналогии с Random Forest — это самая мощная модель, которая использует случайность (Dropout) для повышения устойчивости.

### Как работает
1. Входной слой принимает 8 признаков
2. Первый скрытый слой (256 нейронов) находит множество зависимостей
3. **Dropout (20%)** — случайно отключает 20% нейронов при каждом шаге обучения
4. Второй скрытый слой (128 нейронов) комбинирует паттерны
5. **Dropout (20%)** — снова случайное отключение
6. Третий скрытый слой (64 нейрона) формирует итоговое представление
7. Выходной нейрон выдаёт прочность

### Что такое Dropout
Dropout — это метод регуляризации, при котором во время обучения случайная доля нейронов (20%) отключается на каждом шаге:

```
Без Dropout:     [●] [●] [●] [●] [●]  — все нейроны работают
С Dropout (20%): [●] [○] [●] [●] [○]  — 20% выключены (случайно)
```

На каждом шаге обучения отключаются **разные** нейроны. Это заставляет сеть не полагаться на отдельные нейроны, а распределять знания по всей сети.

### Почему это аналог Random Forest
- **Random Forest** использует случайность при построении деревьев (случайные подвыборки данных и признаков)
- **Dropout** использует случайность при обучении сети (случайное отключение нейронов)
- Оба метода борются с переобучением через **случайность и разнообразие**
- Оба метода — самые мощные в своём классе
- Глубокая архитектура (3 слоя) позволяет находить сложные нелинейные зависимости, как и деревья решений в Random Forest

### Параметры модели
- **Количество параметров:** 43,521
- **Dropout:** 20% после 1-го и 2-го скрытых слоёв
- **Эпох обучения:** 200
- **Размер батча:** 32
- **Оптимизатор:** Adam (lr=0.001)

## Общие компоненты всех сетей

### StandardScaler (масштабирование)
Перед подачей в сеть все признаки масштабируются: из каждого значения вычитается среднее и делится на стандартное отклонение. Без этого признаки с большими значениями (например, цемент ~100-540) доминировали бы над признаками с маленькими (добавка ~0-32).

### Adam (оптимизатор)
Adam — адаптивный оптимизатор, который подстраивает скорость обучения для каждого параметра отдельно. Он сочетает идеи двух алгоритмов: Momentum (учитывает направление предыдущих шагов) и RMSProp (учитывает амплитуду градиентов).

### Validation Split (20%)
При обучении 20% обучающих данных откладывается для валидации. Модель на них не обучается, но после каждой эпохи проверяет свою точность. Это позволяет отслеживать переобучение: если ошибка на обучающих данных падает, а на валидационных растёт — модель начинает запоминать данные вместо обучения.
