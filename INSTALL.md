# Руководство по запуску проекта в VS Code на Windows

Пошаговая инструкция: от скачанного архива до работающего дашборда.

---

## Требования

- **Windows 10/11** (64-бит)
- **Visual Studio Code** (скачать: https://code.visualstudio.com/)
- **Подключение к интернету** (для скачивания Python, библиотек и датасета)
- **~3 ГБ свободного места** на диске (Python + библиотеки + TensorFlow)

---

## Шаг 1. Установка Python 3.12

Нейронные сети работают на TensorFlow, который поддерживает Python 3.9-3.12. Рекомендуется версия **3.12**.

1. Перейдите на сайт: https://www.python.org/downloads/
2. Скачайте **Python 3.12** (не 3.13 и не 3.14 — TensorFlow их не поддерживает)
3. Запустите установщик
4. **ВАЖНО:** на первом экране поставьте галочку **"Add Python to PATH"**
5. Нажмите **"Install Now"**
6. Дождитесь завершения установки

### Проверка

Откройте **Командную строку** (Win + R, введите `cmd`, нажмите Enter):

```cmd
python --version
```

Должно показать:
```
Python 3.12.x
```

Если показывает ошибку — перезагрузите компьютер и попробуйте снова.

---

## Шаг 2. Установка расширения Python в VS Code

1. Откройте **VS Code**
2. Нажмите на иконку расширений на боковой панели (или `Ctrl + Shift + X`)
3. В поиске введите **Python**
4. Установите расширение **"Python"** от Microsoft (первое в списке)
5. Дождитесь завершения установки

---

## Шаг 3. Открытие проекта в VS Code

1. Распакуйте скачанный архив в удобную папку, например:
   ```
   C:\Projects\concrete_prediction
   ```

2. Откройте VS Code
3. Нажмите **Файл → Открыть папку** (или `Ctrl + K`, затем `Ctrl + O`)
4. Выберите папку `C:\Projects\concrete_prediction`
5. Нажмите **"Выбрать папку"**

VS Code откроет проект. На боковой панели вы увидите все файлы:

```

├── INSTALL.md              <- этот файл

│   ├── classic/
│   │   ├── model_linear.py
│   │   ├── model_lasso_ridge.py
│   │   ├── model_random_forest.py
│   │   └── README.md
│   └── neural_network/

```

## Шаг 4. Создание виртуального окружения

Виртуальное окружение изолирует библиотеки проекта от системного Python.

1. Откройте **терминал** в VS Code: меню **Терминал → Новый терминал** (или `` Ctrl + ` ``)
2. Убедитесь, что терминал открыт в папке проекта (должно быть `C:\Projects\concrete_prediction>`)
3. Выполните команду:

```cmd
python -m venv .venv
```

4. VS Code предложит уведомление: **"We noticed a new virtual environment has been created. Do you want to select it for the workspace folder?"** — нажмите **Yes**

Если уведомление не появилось, выберите интерпретатор вручную (следующий шаг).

---

## Шаг 5. Выбор интерпретатора Python

1. Нажмите `Ctrl + Shift + P` (откроется командная палитра)
2. Введите **"Python: Select Interpreter"** и нажмите Enter
3. В списке выберите: **Python 3.12.x ('.venv': venv)**
   (путь будет: `.\.venv\Scripts\python.exe`)

Теперь VS Code использует Python из виртуального окружения.

### Проверка

В нижней строке VS Code (слева) должно отображаться:
```
Python 3.12.x ('.venv': venv)
```

---

## Шаг 6. Установка библиотек

1. Откройте **терминал** в VS Code (`` Ctrl + ` ``)
2. Терминал должен автоматически активировать окружение — в начале строки будет `(.venv)`:
   ```
   (.venv) C:\Projects\concrete_prediction>
   ```
   Если `(.venv)` нет — активируйте вручную:
   ```cmd
   .venv\Scripts\activate
   ```

3. Выполните команду установки:

```cmd
pip install tensorflow pandas numpy matplotlib seaborn scikit-learn openpyxl ucimlrepo dash plotly
```

Установка займёт **5-15 минут** (TensorFlow весит ~500 МБ).

### Проверка

```cmd
python -c "import tensorflow; print('TensorFlow', tensorflow.__version__)"
python -c "import dash; print('Dash', dash.__version__)"
```

Должно показать версии без ошибок.

### Возможные проблемы

| Проблема | Решение |
|---|---|
| `ERROR: No matching distribution found for tensorflow` | У вас Python 3.13+. Установите Python 3.12 и пересоздайте окружение |
| `pip не является внутренней командой` | Окружение не активировано. Выполните `.venv\Scripts\activate` |
| Ошибки при установке numpy/scipy | Выполните `pip install --upgrade pip` и попробуйте снова |
| Терминал не видит `.venv` | Закройте и заново откройте терминал в VS Code |

## Шаг 7. Запуск всех моделей

В терминале VS Code выполните:

```cmd
python run_all.py
```

### Что произойдёт

1. **Предобработка данных** (~10 сек) — скачивание датасета, анализ, визуализация
2. **Линейная регрессия** (~1 сек)
3. **Лассо/гребневая регрессия** (~1 сек)
4. **Random Forest** (~2 сек)
5. **Линейная регрессия NN** (~8 сек) — обучение нейронной сети 200 эпох
6. **Лассо/гребневая регрессия NN** (~8 сек)
7. **Random Forest NN** (~8 сек)

Общее время: **~40 секунд**.

### Ожидаемый вывод

```
✅ Предобработка данных — завершено за 10.2 сек
✅ Линейная регрессия — завершено за 1.3 сек
✅ Лассо/гребневая регрессия — завершено за 1.4 сек
✅ Random Forest — завершено за 1.6 сек
✅ Линейная регрессия NN — завершено за 7.9 сек
✅ Лассо/гребневая NN — завершено за 8.4 сек
✅ Random Forest NN — завершено за 9.2 сек

ВСЕ МОДЕЛИ ЗАВЕРШЕНЫ за 39.8 сек
```

После завершения в проекте появится папка `results/` с графиками, метриками и предсказаниями.

## Шаг 8. Запуск дашборда

В терминале VS Code выполните:

```cmd
python dashboard.py
```

В терминале появится:
```
You can now view your Streamlit app in your browser.
Dash is running on http://127.0.0.1:8050/
```

Браузер откроется автоматически. Если нет — перейдите вручную: **http://localhost:8050**

### Что содержит дашборд

1. Обзор датасета — таблица данных, корреляционная матрица, распределения
2. Сравнительная таблица всех 7 моделей с подсветкой лучшей
3. Графики метрик R2, RMSE, MAE (синий — классика, красный — нейросети)
4. Интерактивные графики предсказаний для любой пары моделей
5. Динамика обучения нейронных сетей по эпохам
6. Архитектуры НС и сравнение с классическими аналогами
7. Таблица проверки предсказаний (первые 20 записей)
8. Радарная диаграмма для выбранных моделей

### Остановка дашборда

Нажмите `Ctrl + C` в терминале VS Code.

## Часто задаваемые вопросы

### VS Code не видит виртуальное окружение
Закройте и заново откройте VS Code. Если не помогло — выберите интерпретатор вручную: `Ctrl + Shift + P` → "Python: Select Interpreter" → `.venv\Scripts\python.exe`.

### Терминал не активирует окружение автоматически
Выполните в терминале вручную:
```cmd
.venv\Scripts\activate
```
Или проверьте настройку VS Code: `Ctrl + Shift + P` → "Python: Select Interpreter" → убедитесь, что выбран `.venv`.

### Красные подчёркивания в коде (import не найден)
Убедитесь, что выбран правильный интерпретатор (`.venv`). Нажмите `Ctrl + Shift + P` → "Python: Select Interpreter" → выберите `.venv`.

### Графики не отображаются в дашборде
Сначала запустите `python run_all.py` — он создаёт все необходимые файлы в `results/`.

### Ошибка "Данные не найдены"
Запустите `python data_preprocessing.py` — он скачивает датасет и подготавливает данные.

### Ошибка "No module named 'tensorflow'"
Убедитесь, что окружение активировано (есть `(.venv)` в начале строки терминала).

### Streamlit не открывает браузер
Перейдите вручную по адресу http://localhost:8050 в любом браузере.

### Как удалить проект полностью?
Просто удалите папку `C:\Projects\concrete_prediction`. Все библиотеки находятся внутри виртуального окружения `.venv`, ничего системного не затронуто.
